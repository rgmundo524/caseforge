# case.yaml v1

`case.yaml` is document metadata used to generate a Source of Funds Notification letter.

## Principles
- Facts and metadata only; no boilerplate narrative text.
- Boilerplate narrative lives in the LaTeX template referenced by `templates.letter_template_id`.
- Generated by CLI wizard (v1).
- No YAML comments required; schema is validated in Rust.

## Deposit scope vs figure scope
- `chains_in_scope` defines **deposit chains only**:
  - these chains require `deposits.csv`
  - these chains are parsed and rendered as bullets
- Figures may depict additional chains beyond deposit chains.
- Requirement: every deposit chain in `chains_in_scope` must be covered by at least one figure's `chains_covered`.

## Top-level keys (required)
- case
- recipient
- investigator
- templates
- stolen_assets
- deposited_assets
- chains_in_scope
- figures
- law_enforcement
- closing

## Swap reality
Perpetrators may swap assets before deposit:
- `deposited_assets.assets` is not required to be a subset of `stolen_assets.assets`.
- Mismatches produce warnings, not hard errors.
