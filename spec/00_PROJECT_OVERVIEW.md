# Source of Funds Notification v1 (Ephemeral Runs)

This folder contains the v1 specifications and starter artifacts for a reproducible **Source of Funds Notification** letter generator.

## v1 goals
- Generate a PDF letter (LuaLaTeX) for a specific recipient VASP.
- Include deposit transaction details **as bullet lists** (always, in v1).
- Include **one or more uploaded figures** (PNG/SVG/PDF), rendered in the order listed in `case.yaml`.
- For each blockchain in scope (deposit chains), accept a deposits CSV export and normalize it for rendering.
- Hyperlink transaction hashes (and deposit addresses when supported) to the correct block explorer.

## v1 execution model (stateless + ephemeral)
- Each execution run is **self-contained** and **run-isolated**.
- Storage is **transient**: inputs and intermediates are held in a temporary work directory and deleted on completion.
- If a document needs regeneration, inputs must be re-submitted.
- A `--keep-workdir` flag preserves the workdir for debugging.

## v1 non-goals
- No web UI (CLI wizard only).
- No API verification calls.
- No non-English output.
- No table mode (bullets only).

## Authoritative inputs
- `case.yaml` (generated by CLI wizard; no boilerplate narrative text)
- `deposits.csv` per deposit chain:
  - `artifacts/<workdir>/chains/<chain_id>/deposits.csv`
- Figures:
  - `artifacts/<workdir>/figures/<figure_id>.<ext>`

## Outputs
Copied to the user-specified `--out` directory:
- Final PDF: `sof_notification_<case_number>.pdf`
- Attachments zip: `attachments.zip` containing:
  - `chains/<chain_id>/deposits.csv` (original uploaded, normalized to deposits.csv)
  - `figures/<figure_id>.<ext>` (uploaded figures)

In the workdir (deleted unless `--keep-workdir`):
- Canonical normalized deposits: `build/canonical/deposits/<chain_id>.csv`
- LaTeX fragments: `report/generated/tables/<chain_id>_deposits_bullets.tex`
- Build manifest: `build/manifest.json`

## Locked v1 decisions
- Deposit details: **always bullet lists**
- `chains_in_scope` is authoritative for **deposit chains only**
- Figures render in YAML order
- Uploaded deposits CSV may have any filename, but is written as **deposits.csv**
- Canonical format: CSV (Parquet later)
- English only (`templates.language = "en"`)
