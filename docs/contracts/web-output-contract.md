# WEB Output Contract

> Status: draft contract

## Purpose

This document defines the current accepted contract for generated WEB outputs.

## Current accepted baseline

A generated WEB output should be:
- a runnable Evidence app root
- bootstrapped using the shared standalone bootstrap seam
- deterministic on rebuild
- linked back to shared case data in `Sources/`

## Expected top-level behavior

After `build-web-draft`, the WEB output should support:
- `npm install`
- `npm run sources`
- `npm run dev`

## Ownership boundary

### Bootstrap-owned runtime root
Expected to remain from bootstrap:
- `package.json`
- `package-lock.json`
- `.npmrc`
- `degit.json`
- runtime scripts

### CaseForge-owned generated content
Expected to be replaced/generated by CaseForge:
- `pages/`
- `sources/`
- `.caseforge/web_output.json`
- connection files pointing at shared case data
- generated narrative/report pages

## Current datasource linkage

The shared DuckDB linkage for case data should live in:

```text
WEB/<output_name>/sources/case/connection.yaml
```

and should point back to:

```text
../../../../Sources/data/case.duckdb
```

relative to that `connection.yaml` location.

## Things the WEB build must not do

- copy the case DuckDB into `WEB/`
- copy raw evidence exports into `WEB/`
- leave starter/demo analysis sources in final case outputs
- make `WEB/` the canonical narrative authoring surface
